## Recorded mechanical choices

1. **Validate mechanism = protovalidate** (`buf.validate` field options).
   The spec mandates machine-readable validate tags with
   descriptor-derived enum bounds but names no technology; protovalidate
   is the maintained buf-native mechanism, its constraints live in the
   descriptor (readable by SPEC-009's test generator), and
   `enum.defined_only = true` satisfies "enum bounds generated from the
   descriptor, never hardcoded" with no generation step to drift.
2. **One proto package `powermanage.v1`**, six service files + common —
   [WIRE-1] shared shapes without cross-package import friction.
3. **ULID rule**: protovalidate **predefined rule** `(powermanage.v1.ulid)`
   extending `buf.validate.StringRules` in common.proto (26 chars,
   Crockford alphabet, per [WIRE-5]); falls back to a documented shared
   `pattern:` constant if predefined-rule extensions fight the remote
   plugins — the fallback is recorded here so using it is not a deviation.
4. **G-1 floor at M1 is the service set** (exactly the six §3.2 names,
   both directions). Empty services mean zero reachable messages — legal
   at M1; the message/field walk runs over whatever exists and the
   liveness fixture proves it can flag. The reachable-message floor
   tightens at M2 when ActionParams lands.
5. **G-2 split**: descriptor half (every enum has `*_UNSPECIFIED = 0`)
   lands now in contract/archtest; the erroring-`default` AST half wires
   up with the first contract-enum `switch` (M3) via the existing
   portable enum-default guard in sdk/guardtest.
6. **G-6 grammar** (syntactic ceiling, recorded): a Go file that imports
   both `encoding/json` and a `…/contract/gen/` package is a violation;
   marshal-site type analysis is out of scope. Escape hatch is an
   identity-keyed allowlist entry with rationale. Population floor: ≥4
   modules walked; liveness via fixtures (bad: both imports; good:
   protojson only).
7. **Generated TS is generated, not compiled**: no node toolchain in the
   verify gate; the TS package's compile/publish flow is SPEC-017's
   (AC-13's "compile" is satisfied for Go; recorded ceiling for TS).
